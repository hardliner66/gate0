# Security Policy

## Reporting a Vulnerability

Please do not open GitHub issues for security vulnerabilities. If you discover a potential security risk, we ask that you report it responsibly via one of the following methods:

**GitHub Private Reporting**: Visit the Security tab of this repository and select "Report a vulnerability".

**Email**: Reach out directly to the maintainer at [your-email].

We value your effort to help keep this project secure and will acknowledge all reports within 48 hours.

## Policy

This project follows a strict "defense in depth" philosophy. While we aim for perfect mechanical safety within the Gate0 engine, we recognize that security is a process. For a detailed breakdown of our threat model and system invariants, please refer to the [Security Model](docs/SECURITY_MODEL.md).
